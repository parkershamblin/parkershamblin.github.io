## Full Article Draft
I am Parker Shamblin, a Computer Science student at the University of South Florida, and this is an overview of the software engineering portfolio I am building in Java, Python, SQL, and computer vision. My goal is simple: create real projects, document them clearly, and show the kind of engineering work I want to do full time.

### Who I am
I focus on backend development, database design, and practical applications of computer vision. I like projects that force me to think about clean architecture, data modeling, and making software understandable for someone who has never seen it before.

### What I build
My current portfolio work centers on:

- Java backend services with clear API and service layers
- SQL schemas that model real workflows and transactions
- Python and OpenCV projects that turn images into structured data

These projects help me practice fundamentals like data validation, error handling, and maintainable code structure, while also showcasing the tools I enjoy using most.

### How I document projects
Good engineering is easier to evaluate when it is well documented. For each project I publish:

- A short problem statement that explains the goal
- An architecture overview that explains the major components
- A database or data flow diagram when it helps clarify structure
- A set of clear steps to run or review the code

This approach helps recruiters and other engineers understand my work quickly, without having to read every line of code first.

### Where to see my work
The best place to see the full set of projects is my portfolio. I also keep my GitHub profile updated with repositories and project READMEs, and I share progress on LinkedIn.

- Portfolio: https://www.parkershamblin.com/
- Projects: https://www.parkershamblin.com/projects
- GitHub: https://github.com/parkershamblin
- LinkedIn: https://www.linkedin.com/in/parkershamblin/

### What I am learning next
I am continuing to deepen my Java backend skills and exploring more advanced computer vision workflows. As I complete new milestones, I will keep this portfolio and my project documentation current.

If you are reviewing my work for a software engineering role, thank you for taking the time to explore it.
